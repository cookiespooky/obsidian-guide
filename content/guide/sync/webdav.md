# Настройка облачной синхронизации Obsidian через Яндекс WebDAV
  
Синхронизация через Яндекс - один из самых простых способов бесплатной синхронизации Obsidian.  
  
Работает независимо от типа устройства:  
  
- Windows  
- macOS  
- Android  
- iOS
  
---
## Настройка

### Шаг 1. Установить плагин Remotely Save в Obsidian

На всех синхронизируемых устройствах.
  
В Obsidian:  
  
Settings → Community Plugins → Turn on community plugins → Browse → Remotely Save → Install → Enable.  
  
---
### Шаг 2. Настроить Яндекс  
  
В Яндексе создать пароль приложения:

- Зайти в пароли приложений:
  https://id.yandex.ru/security/app-passwords
- Выбрать **Файл WebDAV**
- Ввести любое имя
- Скопировать пароль (показывается только 1 раз)
  
---
### Шаг 3. Настроить Obsidian Remotely Save

На всех синхронизируемых устройствах.
  
В Remotely Save выбрать:  
  
Choose A Remote Service: **WebDAV**  

Указать:  
  
- Server Address: **https://webdav.yandex.ru  
- Username: **ваш логин Яндекс** (без `@yandex.ru`)
- Password: **созданный пароль приложения**
- Abort Sync If Modification Above Percentage: **100 (disable the protection)**

Там же в настройках Remotely Save в разделе Check Connectivity можно нажать Check. Если все настроено верно, вы увидите сообщение: `Great! The webdav server can be accessed`.

## Конфликты и подводные камни

#### Одновременное редактирование
Если редактировать один файл одновременно на двух устройствах, либо появятся конфликтные копии, либо конфликтные данные сотрутся.
#### Большие хранилища (>5000 файлов)
Яндекс Диск может медленно индексировать и подвисать при первой синхронизации
  
---
## Когда выбирать Яндекс Диск  
  
Подходит, если:  
  
- Нужен бесплатный способ  
- Не хочется разбираться в S3 или Git  
- Не нужен сложный контроль версий  
- Vault небольшой или средний  
  
---
## Когда лучше выбрать другой способ  
  
- Если нужна история изменений → Git    
- Если хотите без облака → Syncthing    
- Если нужен полный контроль → S3 / MinIO    
  
---
## Архитектурно  
  
Яндекс Диск - это файловая синхронизация, а не протокольная.  
  
Obsidian не знает, что это облако, он просто работает с папкой.  
  
Поэтому:  
  
> Простота = меньше контроля    
> Контроль = больше ответственности

## FAQ - Частые вопросы по синхронизации Obsidian через Яндекс  
  
### Работает ли синхронизация Obsidian через Яндекс бесплатно? 
  
Да. WebDAV-доступ у Яндекс Диска бесплатный.    
Ограничение - объём хранилища на Яндекс Диске в вашем тарифе.  
  
---  
### Нужно ли устанавливать Яндекс Диск как приложение?  
  
Нет.    
В этом сценарии используется WebDAV через плагин Remotely Save.    
Официальное приложение Яндекс Диска не требуется.  
  
---  
### Почему возникает ошибка 401 или 403 при подключении?  
  
Чаще всего причина:  
  
- Используется обычный пароль вместо пароля приложения    
- Неверный логин (нужно указывать без `@yandex.ru`)    
- Ошибка в адресе сервера (должно быть строго: `https://webdav.yandex.ru/`)  
  
---  
### Можно ли использовать основной пароль Яндекс-аккаунта?  
  
Технически да, но не рекомендуется.  
  
Безопаснее создать отдельный пароль приложения для WebDAV.  

---  
### Можно ли синхронизировать несколько vault через один аккаунт?  
  
Да.    
Для каждого vault можно указать разную папку в поле Remote Base Directory в настройках Remotely Save, например:  

/ObsidianWork  
/ObsidianPersonal
  
---  
### Что делать, если появились конфликтные файлы?  
  
1. Найдите файлы с пометкой `conflict` или похожим названием.  
2. Сравните изменения.  
3. Объедините вручную.  
4. Удалите лишнюю копию.  
  
Конфликты обычно возникают при одновременном редактировании.  
  
---  
### Безопасна ли синхронизация через WebDAV?  
  
Передача данных идёт через HTTPS.    
Однако шифрования на стороне клиента нет.  
  
Если требуется дополнительная защита - используйте:  
  
- Obsidian Sync  
- Git + приватный репозиторий  
- Self-hosted сервер с шифрованием  
  
---  
### Подходит ли Яндекс Диск для больших хранилищ (10 000+ файлов)?  
  
Не рекомендуется.  
  
При больших vault возможны задержки, синхронизация может занимать много времени  
  
В таких случаях лучше рассмотреть: Git, S3, Syncthing  
  
---  
### Можно ли использовать Яндекс Диск для командной работы?  
  
Технически да, но не рекомендуется.  
  
WebDAV не предназначен для сложной командной синхронизации.    
Для совместной работы лучше использовать:  Git, Nextcloud  
  
---  
### Что быстрее: WebDAV или S3?  
  
S3 обычно быстрее и стабильнее.    
WebDAV проще в настройке.  
  
Яндекс Диск через WebDAV - это решение проще, но без избыточного контроля.
