# Синхронизация Obsidian через S3

S3 — это облачное объектное хранилище.

Через плагин **Remotely Save** Obsidian может синхронизировать vault напрямую с S3-бакетом.

Этот способ:

- быстрее WebDAV
- стабильнее при больших vault

Работает на всех устройствах:

- Windows
- macOS
- Linux
- Android
- iOS

---

# Когда выбирать S3

S3 подходит, если:

- vault большой (5000+ файлов)
- нужна стабильная синхронизация
- хотите использовать облако без привязки к конкретному сервису

Поддерживаются:

- **AWS S3**
- **Timeweb S3**
- **Beget S3**
- **Yandex Object Storage**
- **Backblaze B2**
- **Cloudflare R2**
- **MinIO**

Фактически любой сервис, который поддерживает **S3 API**.

---

# Настройка

## Шаг 1. Установить плагин Remotely Save

На всех синхронизируемых устройствах.

В Obsidian:

Settings → Community Plugins → Turn on community plugins → Browse → Remotely Save → Install → Enable

---

# Шаг 2. Создать S3-хранилище

Нужно создать **bucket**.

Название может быть любым, например: **obsidian-vault**

Внутри него Remotely Save будет хранить файлы vault.

---

## Пример для Yandex Cloud

Создать bucket: **obsidian-vault**

После создания нужно получить:

- Access Key
- Secret Key
- Endpoint

Endpoint для Yandex:

https://storage.yandexcloud.net

---

# Шаг 3. Настроить Remotely Save

В настройках плагина выбрать:

Choose A Remote Service → S3 or Compatible

Указать параметры:

- Endpoint: **https://storage.yandexcloud.net**
- Region: **ru-central1**
- Bucket Name: **obsidian-vault**
- Access Key ID: **ключ доступа**
- Secret Access Key: **секретный ключ**
- Abort Sync If Modification Above Percentage: **100 (disable the protection)**

Там же в настройках Remotely Save в разделе Check Connectivity можно нажать Check. Если все настроено верно, вы увидите сообщение: `Great! The bucket can be accessed`.

---

# Первая синхронизация

При первой синхронизации:

- все файлы vault загружаются в S3
- может занять несколько минут

После этого синхронизация работает быстро и инкрементально.

---

# Конфликты и подводные камни

### Одновременное редактирование

Как и в [[webdav|WebDAV]], если редактировать один файл на двух устройствах одновременно — появятся конфликтные копии.

---

### Плагины

Некоторые плагины создают временные файлы.

Если их слишком много, синхронизация может стать медленнее.

Обычно это не проблема.

---

### Неверный endpoint

Самая частая ошибка: **Invalid endpoint**

Нужно проверить:

- адрес сервера
- регион
- bucket name

---

# Сколько это стоит

S3 — платный сервис, но для Obsidian расходы обычно минимальные.

Пример: vault 200–500 МБ (это огромный vault)
Стоимость: примерно **20-30 ₽ в месяц**

---

# Архитектурно

S3 работает иначе, чем WebDAV.

WebDAV — это файловая синхронизация. S3 — это **объектное хранилище**.

Файлы хранятся как объекты внутри bucket.
Remotely Save выступает как адаптер:

Obsidian  
↓  
Remotely Save  
↓  
S3 API  
↓  
Object Storage

Поэтому синхронизация:

- быстрее
- стабильнее
- лучше масштабируется

---

# Когда лучше выбрать другой способ

S3 может быть избыточным, если:

- vault небольшой
- хочется максимально простой настройки

В этом случае лучше:

- [[webdav|WebDAV]]
- [[obsidian-sync|Obsidian Sync]]

# FAQ — Частые вопросы по S3 синхронизации

## Нужно ли использовать AWS?

Нет.

Можно использовать любой S3-совместимый сервис:

- Yandex Object Storage
- Timeweb S3
- Beget S3
- Backblaze B2
- Cloudflare R2
- MinIO

---

## Можно ли использовать один bucket для нескольких vault?

Да.

Для этого используется **Remote Prefix**.

Например:

work/  
personal/  
notes/

---

## Насколько это безопасно?

Передача данных происходит через HTTPS.

Доступ контролируется через:

- Access Key
- Secret Key

При необходимости можно:

- ограничить доступ к bucket
- создать отдельного пользователя

---

## Можно ли работать без интернета?

Да.

Obsidian всегда работает локально.

Синхронизация происходит только при подключении к сети.

---

## Подходит ли S3 для больших vault?

Да.

Это одно из лучших решений для:

- 1 000 +
- 10 000+
- 20 000+
- 50 000+ файлов

---

# Итог

S3 — один из самых стабильных способов синхронизации Obsidian.

Он требует немного больше настройки, но:

- быстрее WebDAV
- лучше масштабируется
- работает с большими vault

Для продвинутых пользователей это часто лучший вариант.
