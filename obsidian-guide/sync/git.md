# Синхронизация Obsidian через Git

Git — это система контроля версий, которая позволяет хранить vault Obsidian в репозитории.

Каждое изменение заметок фиксируется как **commit**, поэтому можно:

- видеть историю изменений
- откатываться к прошлым версиям
- синхронизировать vault между устройствами

Git подходит для пользователей, которые хотят **максимальный контроль над своими заметками**.

---

# Когда выбирать Git

Git подходит, если:

- важна история изменений заметок
- хотите полный контроль над данными
- используете Obsidian как систему знаний
- не боитесь немного технической настройки

Git работает на всех устройствах:

- Windows
- macOS
- Linux
- Android
- iOS (требуются дополнительные настройки)

Но мобильная настройка может быть сложнее.

---

# Как работает синхронизация

Vault хранится в **Git-репозитории**.

При изменениях происходит:

изменение заметки  
↓  
commit  
↓  
push  
↓  
репозиторий (GitHub / GitLab / Gitea)  
↓  
pull на другом устройстве

Можно представить это как **историю снимков vault**.

---

# Настройка

## Шаг 1. Создать Git-репозиторий

Можно использовать:

- GitHub
- GitLab
- Bitbucket
- Gitea
- любой self-hosted Git

Создайте новый репозиторий, например: **obsidian-vault**

---

# Шаг 2. Установить плагин Obsidian Git

В Obsidian:

Settings → Community Plugins → Turn on community plugins → Browse → Git → Install → Enable

---

# Шаг 3. Инициализировать репозиторий

В папке vault должен быть Git-репозиторий.

Если он ещё не создан, то в терминале:

```
cd <путь к локальному vault>
git init  
git remote add origin https://github.com/username/obsidian-vault.git
```
где `username` - ваш логин в GitHub.

После этого сделать первый commit:

```
git add .  
git commit -m "initial vault"  
git push
```

---

# Шаг 4. Настроить автоматическую синхронизацию

В настройках **Obsidian Git** можно включить:

- Auto commit
- Auto push
- Auto pull

Обычно используют настройки:

Auto pull on startup - забирает актуальные данные из репозитория
Auto push every 10 minutes - автоматическая синхронизация изменений каждые 10 минут

Теперь синхронизация происходит автоматически.

# Конфликты и подводные камни

### Конфликты изменений

Если один файл изменён на двух устройствах, Git создаёт **merge conflict**.

Это означает, что нужно вручную выбрать версию файла.

Обычно это происходит редко.

---

### Большие бинарные файлы

Git плохо работает с:

- большими PDF
- видео
- архивами

Лучше хранить такие файлы отдельно.

---

### Случайная публикация заметок

Если репозиторий публичный, все заметки будут доступны.

Поэтому рекомендуется:

- использовать **private repository**

---

# Преимущества Git

### История изменений

Можно посмотреть, как менялась заметка.

Например:

- кто изменил
- когда изменил
- что именно изменилось

---

### Откат к прошлой версии

Можно восстановить заметку даже через несколько месяцев.

---

### Надёжность

Git — одна из самых проверенных систем хранения данных.

Используется для хранения исходного кода практически всех современных программ.

---

# Архитектурно

Git хранит изменения как **коммиты**.

Каждый commit — это состояние vault в определённый момент времени.

vault  
↓  
commit  
↓  
repository  
↓  
pull на других устройствах

Это больше похоже на **историю**, чем на обычную синхронизацию.

---

# Когда лучше выбрать другой способ

Git может быть избыточным, если:

- не нужна история изменений
- хочется максимально простой настройки
- используете Obsidian только для личных заметок

В этом случае лучше:

- [[webdav|WebDAV]]
- [[s3|S3]]
- [[obsidian-sync|Obsidian Sync]]

# FAQ — Частые вопросы

### Нужно ли знать Git?

Желательно базовое понимание.

Но большинство операций выполняет плагин **Obsidian Git** автоматически.

---

### Можно ли использовать GitHub бесплатно?

Да.

GitHub позволяет создавать **бесплатные приватные репозитории**.

---

### Можно ли использовать Git для командной работы?

Да.

Git отлично подходит для совместной работы.

Но потребуется договориться о правилах работы с заметками.

---

### Подходит ли Git для больших vault?

Да.

Git хорошо работает с тысячами файлов.

Но большие бинарные файлы могут замедлять репозиторий.

---

### Можно ли синхронизировать без GitHub?

Да.

Можно использовать:

- GitLab
- Gitea
- self-hosted Git сервер

---
